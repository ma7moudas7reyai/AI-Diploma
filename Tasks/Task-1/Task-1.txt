	1- Low Level and High Level?
		Low Level Languages: 
			A)‚úÖ Definition:
				-Low-level languages are very close to the computer‚Äôs hardware. They communicate almost directly with the CPU.

			B)Types of Low-Level Language:
				-Machine Language
						Example:
							10101010 00001111
				-Assembly Language
						Example:
							MOV AX, 5

			C)‚úÖ Advantages:

				-Difficult to debug.

		High Level Languages: 
			A)‚úÖ Definition:

				-Python
				-C++

	2- Interpreted and Compiled:
		Compiled Languages:
			A)‚úÖ Definition:
				-A compiled language is translated all at once into machine code before it runs.
				-This translation is done by a program called a compiler.
				-üîÑ How it works:
					Source Code ‚Üí Compiler ‚Üí Executable File ‚Üí Run
					Once compiled, the program runs directly without needing the compiler again.

			B)üìå Examples:	
				1)C
				2)C++	
					Example (C++):
						#include <iostream>

						int main() {
   							std::cout << "Hello";
						}

			C)‚úÖ Advantages
				1)Faster execution.
				2)Errors are detected before running.
				3)More optimized performance.

			D)‚ùå Disadvantages
				1)Compilation step takes time.
				2)Platform-dependent (you must compile separately for Windows, Linux, etc.).

		Interpreted Languages: 
			A)‚úÖ Definition:
				-An interpreted language is translated line by line during execution.
				-It uses a program called an interpreter.
				üîÑ How it works:
					-Source Code ‚Üí Interpreter ‚Üí Executes line by line
					-There is no separate executable file.

			B)üìå Examples:
				-JavaScript
				-Python
					-Example in Python: 
						print("Hello")

			C)‚úÖ Advantages:
				-Easier debugging (stops at the line with error).
				-Platform-independent (if interpreter exists).
				-Faster development cycle.
			
			D)‚ùå Disadvantages:
				-Slower execution compared to compiled languages.
				-Errors appear during runtime.

			B)üîß Characteristics:

			C)üìå Examples:

				-Java

				-Faster development.

			C)üìå Examples:
				-PHP
				-JavaScript

	4- Open Source and Closed Source:
		Open Source Software:
			A)‚úÖ Definition:
				-Open source software makes its source code publicly available.
				-Anyone can view, modify, and distribute the code according to its license.

			B)üîß Characteristics:
				-Source code is publicly available.
				-Users can modify and improve the software.
				-Community-driven development.
				-Often free (but not always).

			C)üìå Examples:
				-Python
				-Apache HTTP Server
				-Mozilla Firefox
				-Linux
					Example: Linux: Anyone can download the Linux kernel, modify it, and redistribute it under the GPL license.

			D)‚úÖ Advantages:
				-Transparency.
				-Community improvements.
				-High flexibility.
				-Usually free.

			C)‚ùå Disadvantages:
				-Support may depend on community.
				-No guaranteed updates (depends on project).

		Closed Source (Proprietary) Software:
			A)‚úÖ Definition:
				-Closed source software does not provide access to its source code.
				-Only the company or owner can modify it.

			B)üîß Characteristics:
				-Source code is hidden.
				-Controlled by a company.
				-Paid license in many cases.
				-Official support provided by the company.

			C)üìå Examples:
				-Microsoft Windows
				-Microsoft Office
				-Adobe Photoshop

			D)‚úÖ Advantages:
				-Professional support.
				-Stable updates from company.
				-Often polished user experience.

			E)‚ùå Disadvantages:
				-Limited customization.
				-Usually paid.
				-No transparency in internal code.

	5- Languages That Support OOP and Languages That Don't Support OOP:

		üîπ First: What is OOP?
			-OOP (Object-Oriented Programming) is a programming paradigm based on the concept of:
				-Objects
				-Classes
				-Encapsulation
				-Inheritance
				-Polymorphism

		Languages That Support OOP:
			A)‚úÖ Definition:

			B)üìå Examples:
				-C#
				-Java
				-Python
				-C++			
					Example (C++):
						class Car {
							public:
    								string brand;
    								void start() {
       		 							cout << "Car started";
    								}

			C)‚úÖ Characteristics of OOP Languages:

		Languages That Don't Support OOP:
			A)‚úÖ Definition:

				-They usually follow the procedural programming paradigm.

			B)üìå Examples:
				-Pascal
				-C
					Example (C): 
						#include <stdio.h>

						void start() {
    							printf("Car started");
						}

		
		üîπ What Are Procedural Languages?
			-A procedural language is a type of programming language based on the concept of procedure calls (also called functions or routines).
			-In this paradigm, a program is written as a sequence of instructions that are executed step by step.
			-This style is also known as Procedural Programming.
